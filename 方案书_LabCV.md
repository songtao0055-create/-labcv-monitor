# 实验室智能监控与数据管理平台 方案书

---

## 一、项目概述

### 1.1 项目名称
**实验室智能监控与数据管理平台（AILab）**

### 1.2 项目目标
构建一套集视频智能分析、环境安全监测、数据云存储与分析于一体的实验室综合管理平台，覆盖测试台中控室的安全监控、人员行为分析、环境风险预警，以及测试数据的云端存储、权限管理和实时分析。

### 1.3 适用范围
- 测试台中控室
- 耐久测试区域
- 氢气相关测试环境
- 测试数据管理

## 三、功能模块详细设计

### 3.1 视频智能分析模块

#### 3.1.1 陌生人脸识别告警

| 项目 | 说明 |
|------|------|
| **功能描述** | 实时检测进入中控室的人员，与已注册人员人脸库比对，发现陌生人即时告警 |
| **技术方案** | 基于 MTCNN（人脸检测）+ ArcFace（人脸识别） |
| **处理流程** | 摄像头 RTSP 流 → 人脸检测 → 特征提取 → 人脸库比对 → 陌生人判定 → 告警推送 |
| **人脸库管理** | 支持通过小程序/管理端录入授权人员人脸，管理员审核生效 |
| **告警方式** | 飞书推送 + 中控台声光报警器 |

**性能指标：**

- 人脸检测延迟：< 500ms
- 识别准确率：> 99%（LFW 标准）
- 支持同时检测：≥ 5 人

#### 3.1.2 火灾与烟雾识别

| 项目 | 说明 |
|------|------|
| **功能描述** | 通过摄像头画面实时分析火焰、烟雾特征，发现火情立即报警 |
| **技术方案** | YOLOv11 目标检测模型，训练火焰+烟雾专用数据集 |
| **处理流程** | 摄像头 RTSP 流 → 火焰/烟雾检测 → 置信度判定 → 多帧确认 → 告警推送 |
| **防误报机制** | 连续 N 帧（可配置，默认 5 帧）检测到目标才触发报警 |

**检测目标类别：**
- 明火 (flame)
- 烟雾 (smoke)
- 电弧火花 (spark)

#### 3.1.3 氢气浓度联动报警

| 项目 | 说明 |
|------|------|
| **功能描述** | 对接氢气浓度传感器，浓度超标时与视频监控联动，触发报警并自动录像 |
| **数据接入** | 通过 Modbus TCP / RS485 协议读取氢气传感器数据 |
| **报警阈值** | 一级报警：（待定），二级报警：（待定） |
| **联动动作** | 报警器触发 → 摄像头自动截图+录像 → 飞书推送 → 中控台声光报警 |

#### 3.1.4 耐久测试人员瞌睡检测

| 项目 | 说明 |
|------|------|
| **功能描述** | 对耐久测试值班人员进行面部行为分析，检测闭眼、打哈欠、长时间低头等疲劳特征 |
| **技术方案** | MediaPipe Face Mesh + 眼部纵横比 (EAR) + 嘴部纵横比 (MAR)+姿态检测 |
| **判定逻辑** | EAR < 0.2 持续 3 秒 → 闭眼告警；MAR > 0.7 → 打哈欠告警；头部俯仰角 < -30° 持续 5 秒 → 低头告警 |
| **告警方式** | 中控台语音提醒 + 飞书推送 |

**疲劳判定指标：**
| 指标 | 阈值 | 持续触发时间 |
|------|------|-------------|
| 闭眼 (EAR) | < 0.20 | 3 秒 |
| 打哈欠 (MAR) | > 0.70 | 单帧 |
| 点头/低头 | 头部俯仰角 < -30° | 5 秒 |

### 3.2 报警联动模块

#### 3.2.1 中控台报警器

| 项目 | 说明 |
|------|------|
| **硬件方案** | 基于 ESP32 + LED 声光报警器，通过 MQTT/WiFi 与系统通信 |
| **报警等级** | 三级：黄色（注意）、橙色（警告）、红色（紧急） |
| **报警声音** | 内置语音合成芯片，播报具体报警内容（如"中控室有陌生人进入"） |
| **静音控制** | 中控台物理按钮 + 小程序远程静音 |

#### 3.2.2 告警推送策略

| 报警类型 | 等级 | 推送方式 | 推送对象 |
|----------|------|----------|----------|
| 陌生人进入 | 黄色 | 飞书 + 报警器 | 管理员、值班人员 |
| 烟雾检测 | 橙色 | 飞书 + 报警器 | 全员 |
| 明火检测 | 红色 | 飞书 + 报警器 | 全员 + 消防负责人 |
| 氢气浓度超标 | 红色 | 飞书 + 报警器 | 全员 + 安全负责人 |
| 人员瞌睡 | 黄色 | 报警器 + 飞书 | 值班主管 |

### 3.4 测试数据管理模块

#### 3.4.1 云存储方案（云韬共享云端）

| 项目 | 说明 |
|------|------|
| **存储架构** | 对象存储（时序数据）+ 关系数据库（元数据与配置）+ 时序数据库（测试数据） |
| **数据分类** | 常规测试数据、耐久测试数据（高价值）、报警记录、视频片段 |
| **存储周期** | 常规数据：1 年；耐久数据：5 年；报警视频：90 天 |
| **备份策略** | 每日增量备份 + 每周全量备份 |

**推荐技术栈：**
| 组件 | 技术选型 | 用途 |
|------|----------|------|
| 时序数据库 | TDengine / TimescaleDB | 存储测试数据（温度、压力、转速等） |
| 关系数据库 | PostgreSQL | 用户、权限、配置、报警记录 |
| 对象存储 | MinIO / 云韬自有存储 | 视频片段、截图、日志文件 |
| 消息队列 | EMQX / RabbitMQ | 设备数据接入、告警消息分发 |

#### 3.4.2 数据权限管理

| 角色 | 权限范围 |
|------|----------|
| 超级管理员 | 全部数据可见，可管理用户与权限 |
| 测试主管 | 本部门全部测试数据，可导出，不可删除 |
| 测试工程师 | 本人负责的测试项目数据 |
| 访客/审计 | 仅查看脱敏后的统计数据，无导出权限 |
| API 只读 | 第三方系统只读接口，需 Token 认证 |

**权限控制粒度：**
- 项目级：按测试项目隔离数据
- 字段级：敏感字段（如耐久数据原始值）可设置独立访问权限
- 时间级：可设置数据访问有效期

#### 3.4.3 耐久数据实时分析

| 功能 | 描述 |
|------|------|
| **实时采集** | 从测试台 PLC/数采系统获取实时数据流 |
| **异常检测** | 基于统计方法（3σ 原则）+ 滑动窗口检测异常值 |
| **阈值分析** | 自动分析历史数据分布，计算合理阈值区间，标记当前阈值不合理之处 |
| **趋势预测** | 基于时序预测模型（轻量级 LSTM）预测数据趋势，提前发现偏离 |
| **自动报告** | 每日/每周自动生成耐久测试分析报告 |

**阈值合理性分析算法：**
```
1. 统计历史数据分布 (P1, P5, P95, P99)
2. 当前阈值为 [L, H]，计算：
   - 覆盖率 = P(L ≤ x ≤ H) 在历史数据中的占比
   - 如果覆盖率 > 99.9% → 阈值过宽，建议收紧
   - 如果覆盖率 < 95%  → 阈值过窄，建议放宽
   - 正常区间：95% ≤ 覆盖率 ≤ 99.9%
3. 检测到持续超出阈值 → 标记为异常，推送通知
```

## 四、接口设计

### 4.1 摄像头接入

| 协议 | 说明 |
|------|------|
| RTSP | 摄像头视频流拉取 |
| ONVIF | 摄像头 PTZ 控制 |
| 分辨率 | 1080P（主码流）/ 720P（子码流用于分析） |

### 4.2 传感器接入

| 传感器类型 | 协议 | 采集频率 |
|------------|------|----------|
| 氢气浓度传感器 | Modbus RTU / 4-20mA | 1 Hz |
| 温度传感器 | Modbus RTU | 1 Hz |
| 烟雾传感器 | 干接点 / Modbus | 事件触发 |

### 4.3 数据交互接口

| 接口 | 协议 | 方向 | 频率 |
|------|------|------|------|
| 测试数据上传 | MQTT / HTTP | 设备 → 云端 | 1-10 Hz |
| 告警推送 | MQTT | 云端 → 飞书/报警器 | 实时 |
| 视频流 | RTSP → WebRTC | 摄像头 → 云端 → 小程序 | 实时 |
| 控制指令 | MQTT | 小程序 → 报警器 | 按需 |

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 摄像头画面逆光/遮挡 | 识别率下降 | 多角度部署摄像头 + 红外补光 |
| 氢气传感器协议不兼容 | 无法读取数据 | 前期确认传感器型号和协议文档 |
| 网络中断 | 边缘端无法上传数据 | 边缘设备本地缓存，网络恢复后补传 |
| 误报警过多 | 测试人员忽视报警 | 多帧确认机制 + 灵敏度可调节 |
| 本地被恶意关停服务 | 无法检测人员偷懒瞌睡 | 引入心跳检测，检测超过60s发送服务器离线警报 |
| 佩戴安全帽/口罩 | 人脸识别失效 | 支持安全帽检测 + 工牌 RFID 辅助识别 |

## 九、验收标准

| 序号 | 验收项 | 标准 |
|------|--------|------|
| 1 | 陌生人识别 | 识别率 ≥ 90%，识别响应时间 ≤ 2 秒 |
| 2 | 火灾烟雾检测 | 检测率 ≥ 90%，误报率 ≤ 1 次/天 |

| 4 | 瞌睡检测 | 检测率 ≥ 90%，误报率 ≤ 2 次/班次 |
| 5 | 飞书推送 | 报警推送延迟 ≤ 3 秒 |
| 6 | 数据存储 | 数据完整率 ≥ 99.9%，查询响应 ≤ 2 秒 |
| 7 | 阈值分析 | 异常检出率 ≥ 95%，支持自动生成分析报告 |
| 8 | 报警器响应 | 从检测到本地报警 ≤ 1 秒 |
